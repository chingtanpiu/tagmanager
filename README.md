# 标签组合筛选系统

一个基于Python和原生JavaScript的标签管理系统，支持多层级标签分类、条目管理、文件上传和组合筛选功能。

## 功能特点

### 核心功能
- **多层级标签管理**：支持无限层级的标签分类结构
- **条目管理**：创建、编辑、删除条目，支持多种媒体类型
- **组合筛选**：基于标签组合进行精确筛选
- **文件上传**：支持图片、视频、音频、文档等文件类型
- **文件同步删除**：删除非文本类型条目时同步删除uploads文件夹中的关联文件

### 交互功能
- **右键菜单**：支持标签和条目的右键操作
- **批量操作**：支持多选和批量删除、移动
- **拖拽移动**：通过拖拽改变条目的标签归属
- **实时搜索**：在移动条目时提供实时类别搜索
- **框选操作**：支持鼠标框选多个条目

### 技术特色
- **零第三方依赖**：仅使用Python标准库和原生JavaScript
- **现代化UI**：采用CSS Grid和Flexbox布局
- **响应式设计**：适配不同屏幕尺寸
- **完整CRUD**：支持创建、读取、更新、删除操作
- **企业级特性**：配置管理、详细日志、事务处理、完善的错误处理

## 技术架构

### 后端（Python）
- **Web服务器**：基于`http.server`的自定义HTTP服务器
- **数据库**：SQLite数据库，存储标签、条目、关联关系
- **文件处理**：支持Base64编码的文件上传和保存
- **配置管理**：集中式配置文件，支持环境变量
- **日志系统**：详细的操作日志和错误日志记录
- **事务处理**：多步骤操作的事务管理，确保数据一致性
- **错误处理**：完善的错误捕获和详细的错误信息反馈

### 前端（原生JavaScript）
- **DOM操作**：纯原生JavaScript，无框架依赖
- **事件处理**：完整的鼠标、键盘事件处理
- **异步通信**：基于Fetch API的AJAX通信

### 数据库结构
```sql
-- 标签表
tags (id, name, parent_id)

-- 条目表  
items (id, title, content, media_type, version, created_at)

-- 条目标签关联表
item_tags (item_id, tag_id)

-- 版本历史表
version_history (id, table_name, record_id, action, old_data, new_data, timestamp)
```

## 使用说明

### 标签管理
1. **添加大类**：点击"添加大类"按钮创建顶级标签
2. **添加子类**：右键点击标签选择添加子类
3. **删除标签**：右键点击标签选择删除（会同时删除关联条目）
4. **选择标签**：点击标签进行筛选，支持多选组合

### 条目管理
1. **添加条目**：点击"添加条目"按钮
   - 输入标题和内容
   - 选择媒体类型（文本/图片/视频/音频/文档）
   - 非文本类型需要上传文件
   - 选择关联的标签
2. **删除条目**：右键点击条目选择删除，或批量选择后删除
3. **移动条目**：
   - 右键点击条目选择"移动到"
   - 拖拽条目到目标标签
   - 支持批量移动

### 文件处理
- **上传机制**：文件通过Base64编码传输，保存在`uploads/`目录
- **文件命名**：自动生成带时间戳的唯一文件名
- **存储位置**：数据库存储文件路径和元数据，不存储文件内容
- **URL链接**：作为文本内容存储在数据库中，不上传到文件夹
- **文件同步删除**：删除非文本类型条目时同步删除uploads文件夹中的关联文件

### 筛选功能
- **单标签筛选**：点击单个标签
- **多标签组合**：按住Ctrl/Cmd点击多个标签
- **精确匹配**：只显示同时具有所有选中标签的条目

## 开发说明

### 后端API端点
- `GET /api/tags` - 获取标签树结构
- `GET /api/items?tag_ids=1,2,3` - 根据标签筛选条目
- `POST /api/tags` - 创建新标签（包含完整验证）
- `POST /api/items` - 创建新条目（包含事务处理）
- `POST /api/upload` - 上传文件（包含文件名验证）
- `DELETE /api/items` - 批量删除条目（包含事务处理）
- `DELETE /api/tags/:id` - 删除标签
- `PATCH /api/items/move` - 移动条目到不同标签

### 前端主要函数
- `loadTagTree()` - 加载标签树
- `displayItems(items)` - 显示条目列表
- `filterItems()` - 根据选中标签筛选条目
- `handleItemSubmit()` - 处理条目提交
- `moveItemsToTag()` - 移动条目到新标签
- `uploadFile()` - 处理文件上传

### 事件处理
- **标签点击**：筛选功能
- **右键菜单**：上下文操作
- **拖拽操作**：条目移动
- **框选操作**：批量选择
- **文件上传**：异步处理

### 系统配置
- **配置文件**：`config.py` 包含所有系统参数
- **日志系统**：`logger.py` 提供详细日志记录
- **错误处理**：完善的错误捕获和反馈机制
- **事务管理**：确保多步骤操作的数据一致性

## 注意事项

1. **文件存储**：上传文件保存在本地`uploads/`目录，确保有写入权限
2. **数据库**：首次运行会自动创建数据库和表结构
3. **浏览器兼容性**：需要支持现代JavaScript特性（ES6+）
4. **安全性**：当前版本为演示用途，生产环境需要添加安全验证
5. **配置管理**：通过`config.py`文件进行系统配置，支持环境变量覆盖
6. **日志记录**：操作日志保存在`app.log`文件中，定期清理以避免文件过大
